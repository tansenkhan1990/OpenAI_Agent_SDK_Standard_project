# 🐍 Traditional Python & Conda Setup

While `uv` is recommended for speed and modern standards, here is how you can set up and run the OpenAI Agent SDK project using traditional Python virtual environments or Conda.

---

## 🏛 Method 1: Python `venv` (Standard)
The built-in way to isolate your project on macOS/Linux.

### 1. Create a virtual environment
```bash
python3 -m venv .venv
```

### 2. Activate the environment
```bash
source .venv/bin/activate
```

### 3. Install dependencies
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Run the project
```bash
python main.py
```

---

## ❄️ Method 2: Conda / Miniconda
Preferred for cross-platform scientific computing and managing multiple Python versions.

### 1. Create a new environment
```bash
conda create --name agent-sdk python=3.12 -y
```

### 2. Activate the environment
```bash
conda activate agent-sdk
```

### 3. Install dependencies (via pip)
*Note: Conda handles the virtual environment, but the OpenAI SDK should be installed using pip within it.*
```bash
pip install -r requirements.txt
```

### 4. Run the project
```bash
python main.py
```

---

## ⚠️ Important Notes
- **Activation**: You MUST activate your environment (`source .venv/bin/activate` or `conda activate`) EVERY time you open a new terminal window.
- **Deactivation**: Type `deactivate` (for venv) or `conda deactivate` when you're done.
- **Performance**: These methods are traditionally slower and do not use the fast caching or lockfile verification provided by `uv`.

---
*Reference: [Python venv Docs](https://docs.python.org/3/library/venv.html) | [Conda Docs](https://docs.conda.io/)*
